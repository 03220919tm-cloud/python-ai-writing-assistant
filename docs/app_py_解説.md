# `app.py` を理解しよう

## このファイルの役割

`app.py` はアプリの**玄関口（トップページ）**です。機能を持つページではなく、アプリ全体の入口として「ようこそ画面」を表示するだけのシンプルなファイルです。

---

## コードを1行ずつ読む

```python
import streamlit as st
```

Streamlit ライブラリを読み込んでいます。`st` という短い名前で使えるようにしています。

---

```python
st.set_page_config(
    page_title="AI Writing Tool",
    page_icon="✍️",
    layout="wide",
    initial_sidebar_state="expanded",
)
```

ブラウザタブに表示される**タイトルとアイコン**、画面レイアウト（`wide` = 横幅いっぱい）、サイドバーを最初から開いた状態にする設定です。**必ずページの一番最初に呼ぶ必要があります。**

---

```python
st.title("✍️ AI ライティングツール")
st.markdown("---")
```

画面上部に大見出しと区切り線を表示します。

---

```python
st.markdown("""
### ようこそ！
...
""")
```

Markdown 形式でページ本文を書いています。表・コードブロック・リンクはすべて Markdown の記法で書かれています。

---

```python
st.sidebar.success("上のメニューから機能を選択してください")
```

左側のサイドバーに緑色のメッセージを表示します。`st.sidebar.〇〇` とすることで、メイン画面ではなくサイドバーに要素を置けます。

---

## なぜ `pages/` フォルダが必要なのか

Streamlit には**マルチページ機能**があり、`pages/` フォルダにファイルを置くだけで自動的にサイドバーにメニューが追加されます。

```
app.py            ← トップページ（今見ているファイル）
pages/
  1_blog.py       ← 📝 ブログ記事
  2_email.py      ← 📧 メール返信
  3_scheduler.py  ← 📅 スケジューラー
  4_proofread.py  ← ✏️ 文章校正
  5_sns.py        ← 📱 SNS投稿
  6_summary.py    ← 📋 要約
```

ファイル名の先頭の数字（`1_`, `2_`）が**表示順**を決めています。

---

## まとめ

| コード | 役割 |
|--------|------|
| `set_page_config()` | タブタイトル・レイアウト設定 |
| `st.title()` | 大見出し |
| `st.markdown()` | 本文・表・リンクなど |
| `st.sidebar.〇〇` | サイドバーへの要素配置 |

`app.py` 自体はとてもシンプルで、**実際の機能はすべて `pages/` の各ファイルに分かれています。**
次は `pages/1_blog.py` を見ると、AIとの連携部分（プロンプト構築・ストリーミング表示）が理解できます。
